### 3. Step-by-Step Implementation Instructions

#### STEP 1: Establish the Calculation Engine (`src/tools.py`)
- Do NOT let the LLM calculate percentages or variances natively. Write deterministic Pandas functions.
- Handle Financial Sign Conventions carefully:
  - According to `accounting_policy_notes.md`, costs/expenses are represented as negative numbers.
  - Formula for Revenue Items: Variance = Actual - Target (Positive = Favorable)
  - Formula for Expense Items (Negative amounts): Variance = Target - Actual (Or handle it such that an actual spend of -150 vs a budget of -100 properly reflects an Unfavorable variance of -50).
- Expose a main function: `get_variance_report(month: str, baseline: str)` returning calculated differences, percentage variances, and the top 2-3 variance drivers sorted by absolute impact magnitude.

#### STEP 2: Establish the Local Knowledge Store (`src/rag.py`)
- Load `finance_glossary.md` and `accounting_policy_notes.md`.
- Vectorize these files into a local semantic index using ChromaDB or FAISS.
- Create a helper tool `query_kb(query: str)` so that the drafting pipeline can query classification definitions and narrative style formatting rules before finalizing text.

#### STEP 3: Implement the Dual-State Orchestration Engine (`src/engine.py`)
Build a pipeline that operates in a 4-stage sequential agent flow:
1. **Planner**: Identifies the selected month and requested comparison baseline.
2. **Compute**: Triggers `src/tools.py` to extract calculated tables and top drivers.
3. **Draft**: Queries RAG for definitions, then passes data to the LLM to write an executive commentary according to the style rules in `finance_glossary.md` (State the month/baseline, explicitly quantify drivers with INR lakhs and % variance, call out top 2-3 drivers max).
4. **Review Mode Execution**: 
   - Pass the initial draft to a secondary validation prompt layer. 
   - This layer acts as a strict auditor flagging weak assumptions, unsupported market claims (e.g., explaining drops by blaming "competitors" when data does not state it), and missing variables.
   - It appends a dedicated "Review Mode Output" section listing flagged risks and asking clarifying questions.

#### STEP 4: Build the UI Dashboard (`app.py`)
Develop a clean multi-tab Streamlit dashboard allowing users to:
- Select a reporting month via a drop-down menu (e.g., 2025-09, 2025-11).
- Select a target baseline via a radio button (Actual vs Budget or Actual vs Forecast).
- Display insights across three functional layout tabs:
  - **Tab 1: Calculated Insights**: Shows data tables and top drivers calculated via Pandas.
  - **Tab 2: Copilot Narrative Draft**: Shows the beautifully formatted Markdown executive summary.
  - **Tab 3: Review Mode Audit Trail**: Displays the flagged assumptions and clarifying questions generated by the validator layer.

#### STEP 5: Automated Evaluation Framework
- Build a utility script that reads each prompt row in `evaluation_prompts.xlsx`.
- Run the agent engine across each row and save outputs to a tracking log file.
- Use an LLM-as-a-judge prompt to confirm if the output correctly captures the target metrics listed under `expected_focus` without fabricating extraneous figures.

### 4. Technical Constraints
- Zero Fabricated Numbers: If a metric or rationale is not inside the data or RAG context, state "Data Insufficient" in Review Mode.
- Code Safety: Ensure all spreadsheet ingestion reads Excel sheets using `pd.read_excel(filepath, sheet_name='in')`.
